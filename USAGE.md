# 3D Model Viewer 使用指南

## 快速开始

### 1. 启动应用

**macOS/Linux:**
```bash
./start.sh
```

**Windows:**
```cmd
start.bat
```

或直接使用 Wails CLI：
```bash
wails dev
```

### 2. 加载 3D 模型

1. 应用启动后，点击 "📁 Select 3D Model" 按钮
2. 在文件选择器中选择一个 3D 模型文件
3. 模型将自动加载并显示在查看器中

### 3. 交互操作

- **旋转模型**: 按住鼠标左键并拖动
- **缩放视图**: 滚动鼠标滚轮
- **平移视图**: 按住鼠标右键并拖动

## 支持的文件格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| OBJ | .obj | Wavefront 3D 对象文件格式 |
| GLB | .glb | 二进制 glTF 格式（包含纹理） |
| GLTF | .gltf | JSON 格式的 glTF 文件 |
| STL | .stl | 3D 打印常用的立体光刻格式 |

## 功能说明

### 模型信息显示

加载模型后，应用会显示：
- 文件格式类型
- 文件路径
- 模型状态

### 查看器特性

- **自动居中**: 模型加载后自动居中显示
- **自适应缩放**: 根据模型大小自动调整视图
- **网格参考**: 显示底部网格，便于空间定位
- **光照效果**: 包含环境光、方向光和背光，提供良好的视觉效果

## 常见问题

### Q: 模型无法加载？

A: 请确保：
1. 文件格式受支持（OBJ、GLB、GLTF、STL）
2. 文件没有损坏
3. 文件大小不超过系统内存限制

### Q: 模型显示不正确？

A: 可能是模型坐标系的问题，尝试：
1. 重新导出模型文件
2. 检查模型的缩放和旋转设置
3. 尝试不同的文件格式

### Q: 性能问题？

A: 优化建议：
1. 降低模型的面数
2. 使用 GLB 格式（通常更高效）
3. 关闭其他占用 GPU 的应用

### Q: 开发模式无法启动？

A: 检查：
1. 是否安装了所有依赖（Go、Node.js、Wails CLI）
2. 网络连接是否正常（需要下载依赖）
3. 端口 34115 是否被占用

## 开发说明

### 项目结构

```
3dview/
├── app.go                 # Go 后端逻辑
├── main.go               # Wails 入口
├── frontend/
│   ├── src/
│   │   ├── App.jsx       # 主组件
│   │   ├── App.css       # 样式
│   │   └── main.jsx      # React 入口
│   └── package.json      # 前端依赖
└── build/                # 构建输出
```

### 自定义样式

编辑 `frontend/src/App.css` 文件来修改：
- 颜色主题
- 布局
- 字体
- 组件样式

### 添加新功能

1. **后端功能**: 在 `app.go` 中添加新的方法
2. **前端功能**: 在 `frontend/src/App.jsx` 中添加组件
3. 重新生成绑定：运行 `wails generate module`
