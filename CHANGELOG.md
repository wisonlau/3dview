## v1.0.3 (2026-04-15)

### 🔧 修复 (Fixes)

#### WebGL 上下文清理问题
- **问题**: 选择第二个文件时出现 `TypeError: null is not an object (evaluating 'gl.getShaderPrecisionFormat(gl.VERTEX_SHADER, gl.HIGH_FLOAT).precision')`
- **原因**: WebGL 上下文没有完全清理，在同一 canvas 上重复创建 WebGLRenderer 导致上下文冲突
- **解决方案**:
  - 彻底清理场景中的所有对象（几何体、材质、光源）
  - 清理 OrbitControls
  - 使用 canvas 替换策略，每次加载创建新的 canvas
  - 完整的资源释放和内存清理

#### 技术改进
- **对象清理**: 遍历场景树，清理所有对象和资源
- **Canvas 替换**: 避免 WebGL 上下文冲突
- **清理顺序**: 按正确顺序清理，避免资源泄漏
- **内存管理**: 改善内存使用，无内存泄漏

## v1.0.2 (2026-04-15)

### 🔧 修复 (Fixes)

#### STLLoader 加载问题
- **问题**: `undefined is not a constructor (evaluating 'new window.THREE.STLLoader()')`
- **原因**: CDN 路径不正确，STLLoader 无法正确加载
- **解决方案**: 
  - 移除所有 CDN 依赖
  - 改用 npm 模块导入 Three.js
  - 使用 ES6 模块语法导入所有 Three.js 组件

#### 技术改进
- **模块化**: 使用 `import * as THREE from 'three'` 和专用加载器导入
- **版本兼容性**: 确保 Three.js 及其组件版本一致
- **构建优化**: Vite 自动优化和压缩代码
- **包大小**: ~706 KiB (未压缩), ~191 KiB (gzip 压缩)

## v1.0.1 (2026-04-15)

### 🔧 修复 (Fixes)

#### STL 文件加载问题
- **问题**: STL 文件显示 "Failed to load 3D model" 错误
- **原因**: 
  1. STL 文件被作为文本读取而不是二进制格式
  2. 后端尝试检查文件是否存在，但浏览器安全限制阻止
- **解决方案**:
  - 修改文件读取逻辑，STL 文件使用 ArrayBuffer 读取
  - 简化后端验证逻辑，只检查文件扩展名
  - 添加详细的错误日志和验证
